========================================
✅ COMPLETE AUTO-START SETUP
========================================

Your HMS server will now start automatically on every boot!

========================================
WHAT'S CONFIGURED:
========================================

1. ✅ HMS Startup Script
   - File: startup_hms.bat
   - Waits for Docker Desktop (up to 2 minutes)
   - Starts all HMS services automatically
   - Location: Windows Startup folder

2. ✅ Windows Startup Shortcut
   - File: HMS_AutoStart.lnk
   - Runs automatically on login
   - Location: Startup folder

3. ⚠️  Docker Desktop Auto-Start
   - Needs to be enabled manually (see below)

========================================
FINAL STEP: Enable Docker Desktop Auto-Start
========================================

IMPORTANT: You need to enable this in Docker Desktop:

1. Open Docker Desktop
   (Run: ENSURE_DOCKER_AUTOSTART.bat to open it)

2. Click Settings (gear icon, top right)

3. Go to "General" tab

4. Check "Start Docker Desktop when you log in"

5. Click "Apply & Restart"

That's it! Docker Desktop will now start automatically.

========================================
HOW IT WORKS:
========================================

On System Boot:
1. Windows starts
2. Docker Desktop starts automatically (if enabled)
3. HMS startup script runs (from Startup folder)
4. Script waits for Docker Desktop to be ready
5. HMS services start automatically
6. Server ready at http://192.168.0.102:8000

Total Time: 2-3 minutes after boot

========================================
TO TEST:
========================================

1. Enable Docker Desktop auto-start (see above)
2. Restart your computer
3. Wait 2-3 minutes
4. Check services: docker-compose ps
5. Access: http://192.168.0.102:8000

If services are running, auto-start is working! ✅

========================================
VERIFY AUTO-START:
========================================

Check Startup Folder:
- Press Win+R
- Type: shell:startup
- Press Enter
- You should see: HMS_AutoStart.lnk

Check Docker Desktop Settings:
- Docker Desktop -> Settings -> General
- "Start Docker Desktop when you log in" should be checked

========================================
TO DISABLE AUTO-START:
========================================

1. Docker Desktop:
   - Settings -> General
   - Uncheck "Start Docker Desktop when you log in"

2. HMS Services:
   - Delete: HMS_AutoStart.lnk from Startup folder
   - Or disable in Task Scheduler

========================================
TROUBLESHOOTING:
========================================

Services not starting after boot?
→ Check Docker Desktop auto-start is enabled
→ Check Startup folder has HMS_AutoStart.lnk
→ Check startup_log.txt for errors

Docker Desktop not starting?
→ Enable in Docker Desktop settings
→ Check Windows Services for PostgreSQL

Services start but can't access?
→ Run allow_port_8000_firewall.bat as Administrator
→ Check: docker-compose ps
