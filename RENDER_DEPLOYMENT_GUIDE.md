## ✅ Prerequisites

Before you begin, make sure you have:

## 🚀 Quick Start

### Option 1: One-Click Deploy (Recommended)

1. **Push to GitHub**
   ```bash
   git add .
   git commit -m "Prepare for Render deployment"
   git push origin main
   ```

2. **Connect Render to GitHub**
   - Go to [dashboard.render.com](https://dashboard.render.com)
   - Click "New" → "Blueprint"
   - Connect your GitHub account
   - Select your HMS repository
   - Render will automatically detect `render.yaml` and create all services

3. **That's it!** Render will automatically:
   - Create PostgreSQL database
   - Create Redis instance
   - Deploy web service
   - Deploy Celery workers
   - Set up automatic deployments

### Option 2: Manual Setup

If you prefer manual control, follow the [Detailed Setup Instructions](#detailed-setup-instructions) below.

## 📚 Detailed Setup Instructions

### Step 1: Prepare Your Repository

1. **Ensure all files are committed:**
   ```bash
   git status
   git add .
   git commit -m "Add Render configuration files"
   git push origin main
   ```

2. **Verify these files exist in your repository:**
   - ✅ `render.yaml` - Service configuration
   - ✅ `build.sh` - Build script
   - ✅ `Dockerfile` - Container configuration
   - ✅ `requirements.txt` - Python dependencies
   - ✅ `env.render.example` - Environment variables template

### Step 2: Create Render Account

1. Go to [render.com](https://render.com)
2. Sign up with GitHub (recommended) or email
3. Verify your email address

### Step 3: Connect GitHub Repository

1. In Render Dashboard, click "New" → "Blueprint"
2. Click "Connect Account" under GitHub
3. Authorize Render to access your repositories
4. Select your HMS repository
5. Click "Connect"

### Step 4: Deploy Blueprint

Render will automatically detect `render.yaml` and show you:

- **Web Service**: `hms-web` (Django application)
- **PostgreSQL Database**: `hms-db`
- **Redis**: `hms-redis`
- **Celery Worker**: `hms-celery-worker`
- **Celery Beat**: `hms-celery-beat`

Click **"Apply"** to create all services.

### Step 5: Configure Environment Variables

After deployment starts, you need to set sensitive environment variables:

#### A. Web Service (`hms-web`)

Go to `hms-web` → Environment → Add Environment Variables:

| Variable | Value | Example |
|----------|-------|---------|
| `ALLOWED_HOSTS` | Your domain(s) | `hms-web.onrender.com,yourdomain.com` |
| `SITE_URL` | Your production URL | `https://hms-web.onrender.com` |
| `CORS_ALLOWED_ORIGINS` | Frontend URLs | `https://yourdomain.com` |
| `EMAIL_HOST` | SMTP server | `smtp.gmail.com` |
| `EMAIL_HOST_USER` | Your email | `your-email@gmail.com` |
| `EMAIL_HOST_PASSWORD` | App password | `your-app-specific-password` |
| `SMS_API_KEY` | SMS Notify key | `your-sms-api-key` |

**Important Notes:**

- `SECRET_KEY` is auto-generated by Render (don't change)
- `DATABASE_URL` is auto-set from PostgreSQL service
- `REDIS_URL` is auto-set from Redis service

#### B. Worker Services

Environment variables are automatically inherited from the web service.

### Step 6: Monitor Deployment

1. Go to "Logs" tab for each service
2. Watch for successful deployment messages:
   ```
   ✅ Build completed successfully!
   Starting gunicorn...
   Listening at: http://0.0.0.0:10000
   ```

3. Check deployment status:
   - 🟢 Green = Live
   - 🟡 Yellow = Building
   - 🔴 Red = Failed (check logs)

## 🔧 Post-Deployment Configuration

### 1. Create Superuser

**Option A: Using Render Shell**
```bash
# In Render Dashboard, go to hms-web → Shell
python manage.py createsuperuser
```

**Option B: Uncomment in build.sh**
Edit `build.sh` and uncomment the superuser creation section, then redeploy.

### 2. Verify Health Check

Visit: `https://your-app.onrender.com/health/`

You should see: `{"status": "ok"}`

### 3. Access Admin Panel

Visit: `https://your-app.onrender.com/admin/`

Login with your superuser credentials.

### 4. Test Core Features

- ✅ Login works
- ✅ Database queries work
- ✅ Static files load
- ✅ Media uploads work (if configured)
- ✅ Email notifications send
- ✅ SMS notifications send

### 5. Configure Custom Domain (Optional)

1. In Render Dashboard, go to `hms-web` → Settings → Custom Domains
2. Click "Add Custom Domain"
3. Enter your domain (e.g., `hms.yourdomain.com`)
4. Add the provided DNS records to your domain registrar:
   ```
   Type: CNAME
   Name: hms
   Value: hms-web.onrender.com
   ```
5. Wait for DNS propagation (5-30 minutes)
6. Render will automatically provision SSL certificate

## 🐛 Troubleshooting

### Build Failures

**Problem**: Build fails with dependency errors

**Solution**:
```bash
# Check requirements.txt has all dependencies
pip freeze > requirements.txt
git commit -am "Update requirements"
git push
```

**Problem**: Static files not collecting

**Solution**:
```python
# Verify settings.py has:
STATIC_ROOT = BASE_DIR / 'staticfiles'
STATICFILES_STORAGE = 'whitenoise.storage.CompressedManifestStaticFilesStorage'
```

### Database Issues

**Problem**: Database connection fails

**Solution**:
1. Check `DATABASE_URL` is set correctly
2. Verify PostgreSQL service is running (green status)
3. Check database logs for errors

**Problem**: Migrations fail

**Solution**:
```bash
# In Render Shell:
python manage.py migrate --run-syncdb
python manage.py migrate --fake-initial
```

### Redis Connection Issues

**Problem**: Redis connection timeout

**Solution**:
1. Verify `REDIS_URL` environment variable
2. Check Redis service status
3. Make sure Redis and Web service are in same region

### Performance Issues

**Problem**: App is slow

**Solutions**:
1. **Upgrade Plan**: Free tier has limitations
   - Go to Service → Settings → Upgrade
   
2. **Enable Query Optimization**:
   ```python
   # In settings.py
   DATABASES['default']['CONN_MAX_AGE'] = 600
   ```

3. **Enable Redis Caching**:
   ```
   USE_REDIS_CACHE=True
   ```

4. **Scale Workers**:
   - Increase worker concurrency in `render.yaml`

### Email Not Sending

**Problem**: Emails not being sent

**Solutions**:

1. **Gmail**: Use App-specific password
   - Go to Google Account → Security → App Passwords
   - Generate password for "Mail"
   - Use that password in `EMAIL_HOST_PASSWORD`

2. **Verify Settings**:
   ```python
   EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
   EMAIL_USE_TLS = True
   EMAIL_PORT = 587
   ```

3. **Test Email**:
   ```bash
   # In Render Shell
   python manage.py shell
   from django.core.mail import send_mail
   send_mail('Test', 'This is a test', 'from@example.com', ['to@example.com'])
   ```

## 💰 Cost Optimization

### Free Tier Limitations

Render's free tier includes:
- ✅ 750 hours/month web service
- ✅ PostgreSQL: 1GB storage, expires after 90 days
- ✅ Redis: 25MB storage
- ❌ Services spin down after 15 min inactivity
- ❌ 500 build minutes/month

### Optimization Tips

#### 1. Prevent Service Spin-Down

**Option A**: Keep-alive ping service (Free)
- Use UptimeRobot or similar service
- Ping your app every 10 minutes

**Option B**: Upgrade to Starter plan ($7/month)
- Services stay active 24/7
- Much faster response times

#### 2. Optimize Build Times

```bash
# In Dockerfile, use pip cache
RUN --mount=type=cache,target=/root/.cache/pip \
    pip install -r requirements.txt
```

#### 3. Database Optimization

- **Free PostgreSQL expires after 90 days**
- Consider upgrading to Starter ($7/month) for persistent database
- Or migrate to external PostgreSQL provider

#### 4. Reduce Worker Costs

If you don't need background tasks immediately:
- Comment out Celery services in `render.yaml`
- Re-enable when needed

### Recommended Production Setup ($21/month)

```
Web Service (Starter):     $7/month
PostgreSQL (Starter):      $7/month
Redis (Starter):           $7/month
Celery Worker (Starter):   $0/month (same as web)
--------------------------------------------
Total:                     ~$21/month
```

## ⚙️ Advanced Configuration

### 1. Enable Auto-Deploy on Push

Already configured! Every push to `main` branch triggers automatic deployment.

To disable:
```yaml
# In render.yaml
services:
  - type: web
    autoDeploy: false  # Change to false
```

### 2. Environment-Based Deployments

Create multiple environments:

```bash
# Preview environment
git checkout -b staging
git push origin staging

# In Render, create new Blueprint pointing to 'staging' branch
```

### 3. Database Backups

**Manual Backup**:
```bash
# In Render Shell
pg_dump $DATABASE_URL > backup.sql
```

**Automatic Backups**:
- Upgrade to Standard plan ($21/month)
- Daily automatic backups included

### 4. Monitoring & Logging

#### Enable Sentry (Error Tracking)

1. Sign up at [sentry.io](https://sentry.io)
2. Create new Django project
3. Add to Render environment:
   ```
   SENTRY_DSN=https://your-sentry-dsn@sentry.io/project-id
   ```

#### View Logs

```bash
# Real-time logs in Dashboard
Go to Service → Logs → Enable Live Tail

# Download logs
Go to Service → Logs → Download
```

### 5. CI/CD Integration

Already configured! Your `render.yaml` handles:
- ✅ Automatic builds on push
- ✅ Health checks
- ✅ Zero-downtime deployments

### 6. Static Files with CDN (Optional)

For faster static file delivery:

1. **Use Cloudflare CDN**:
   - Add your domain to Cloudflare
   - Enable CDN
   - Static files automatically cached

2. **Or use AWS S3/CloudFront**:
   ```python
   # settings.py
   STATICFILES_STORAGE = 'storages.backends.s3boto3.S3StaticStorage'
   AWS_STORAGE_BUCKET_NAME = 'your-bucket'
   ```
