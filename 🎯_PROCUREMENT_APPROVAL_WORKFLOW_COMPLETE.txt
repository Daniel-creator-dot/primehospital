======================================================================
                        🎊 HOW THE WORKFLOW WORKS
======================================================================

ROLE 1: PROCUREMENT STAFF (Requesters)
---------------------------------------
What they do:
  1. Create procurement requests
  2. Add items and quantities
  3. Submit for approval
  
Access:
  - Dashboard: /hms/procurement/approval/dashboard/
  - Create: /hms/procurement/approval/create/
  
Status: "Draft" → "Submitted"


ROLE 2: ADMINISTRATORS (First Approval)
----------------------------------------
What they do:
  1. Review submitted requests
  2. Check necessity and budget
  3. Approve or reject
  
Access:
  - Pending list: /hms/procurement/admin/pending/
  - Approve: /hms/procurement/admin/{id}/approve/
  - Reject: /hms/procurement/admin/{id}/reject/
  
Status: "Submitted" → "Admin Approved"


ROLE 3: ACCOUNTS DEPARTMENT (Final Approval + Magic!)
------------------------------------------------------
What they do:
  1. Review admin-approved requests
  2. Verify budget compliance
  3. Approve → AUTOMATIC ACCOUNTING!
  
Access:
  - Pending list: /hms/procurement/accounts/pending/
  - Approve: /hms/procurement/accounts/{id}/approve/
  - Reject: /hms/procurement/accounts/{id}/reject/
  
Status: "Admin Approved" → "Accounts Approved"

AUTOMATIC CREATION:
  ✅ Accounts Payable entry
  ✅ Expense entry
  ✅ Payment Voucher
  
All created automatically when accounts approves!

======================================================================
                        🔑 ACCESS CONTROL & PERMISSIONS
======================================================================

PROCUREMENT STAFF:
  Can:
    - Create requests
    - View own requests
    - Submit requests
  Cannot:
    - Approve requests
    - Access accounting

ADMINISTRATORS:
  Can:
    - View all submitted requests
    - Approve/reject at admin level
    - Add comments
  Cannot:
    - Access accounts approval
    - Create accounting entries
  
  Permission: can_approve_procurement_admin

ACCOUNTS STAFF:
  Can:
    - View admin-approved requests
    - Approve/reject at accounts level
    - Trigger accounting entry creation
    - View accounting impact
  Cannot:
    - Skip admin approval
  
  Permission: can_approve_procurement_accounts

FINANCE STAFF:
  Can:
    - Process payments
    - View payment vouchers
    - Mark as paid
  Cannot:
    - Bypass approval workflow

======================================================================
                        💡 HOW TO USE (STEP BY STEP)
======================================================================

FOR PROCUREMENT STAFF TO CREATE REQUEST:
-----------------------------------------
1. Go to: http://127.0.0.1:8000/hms/procurement/approval/dashboard/

2. Click "Create New Request"

3. Fill in:
   - Priority (Low/Normal/High/Urgent)
   - Justification
   - Items (name, quantity, price)
   - Notes

4. Click "Save as Draft"

5. Review the request

6. Click "Submit for Approval"

7. Request sent to administrators!


FOR ADMINISTRATORS TO APPROVE:
-------------------------------
1. Go to: http://127.0.0.1:8000/hms/procurement/admin/pending/

2. See list of submitted requests

3. Click on a request to review

4. Check:
   - Items needed
   - Quantities
   - Estimated costs
   - Justification

5. Decision:
   - To Approve: Click "Approve", add comments, submit
   - To Reject: Click "Reject", give reason, submit

6. If approved: Request moves to accounts!


FOR ACCOUNTS TO APPROVE (CREATES ACCOUNTING!):
-----------------------------------------------
1. Go to: http://127.0.0.1:8000/hms/procurement/accounts/pending/

2. See list of admin-approved requests

3. Click on a request to review

4. Check:
   - Budget availability
   - Vendor information
   - Total amount
   - Admin comments

5. Click "Approve"

6. Add comments (optional)

7. Submit

8. ✨ MAGIC HAPPENS:
   - System creates Accounts Payable
   - System creates Expense Entry
   - System creates Payment Voucher
   - All automatically!

9. See success message with entry numbers

10. Check accounting:
    - Go to: /admin/hospital/accountspayable/
    - Go to: /admin/hospital/expense/
    - Go to: /admin/hospital/paymentvoucher/

======================================================================
                        ⚙️ SETUP PERMISSIONS
======================================================================

To set up user permissions, run in Django shell:

```python
from django.contrib.auth.models import Permission, User
from django.contrib.contenttypes.models import ContentType
from hospital.models_procurement import ProcurementRequest

# Get content type
ct = ContentType.objects.get_for_model(ProcurementRequest)

# Create permissions if not exist
admin_perm, _ = Permission.objects.get_or_create(
    codename='can_approve_procurement_admin',
    name='Can approve procurement at admin level',
    content_type=ct,
)

accounts_perm, _ = Permission.objects.get_or_create(
    codename='can_approve_procurement_accounts',
    name='Can approve procurement at accounts level',
    content_type=ct,
)

# Assign to users
admin_user = User.objects.get(username='admin')
admin_user.user_permissions.add(admin_perm, accounts_perm)

accounts_user = User.objects.get(username='accounts_manager')
accounts_user.user_permissions.add(accounts_perm)
```
