========================================
IF STILL NOT WORKING:
========================================

1. CHECK FIREWALL (Most Common Issue)
   --------------------------------
   Run as Administrator: FIX_NETWORK_ACCESS.bat
   
   OR manually:
   - Windows Defender Firewall -> Advanced Settings
   - Inbound Rules -> New Rule
   - Port -> TCP -> 8000 -> Allow
   - Name: "HMS Docker Port 8000"

2. CHECK FROM ANOTHER DEVICE
   --------------------------
   - Make sure device is on same WiFi network
   - Try: http://192.168.0.102:8000
   - If connection refused: Firewall issue
   - If 400 Bad Request: ALLOWED_HOSTS issue (restart services)

3. CHECK ROUTER SETTINGS
   ---------------------
   - Some routers block device-to-device communication
   - Check "AP Isolation" or "Client Isolation" settings
   - Disable if enabled

4. CHECK ANTIVIRUS
   ----------------
   - Some antivirus may block port 8000
   - Add exception for Docker/port 8000

5. TEST LOCAL ACCESS FIRST
   ------------------------
   From your computer: http://localhost:8000
   - If this works: Network/firewall issue
   - If this doesn't work: Service issue

========================================
COMMON ERRORS:
========================================

"Connection refused"
→ Firewall blocking (run FIX_NETWORK_ACCESS.bat as admin)
→ Services not running (docker-compose up -d)

"400 Bad Request" or "Disallowed Host"
→ Restart services: docker-compose restart web
→ Check ALLOWED_HOSTS in docker-compose.yml

"Can't connect" from network but works locally
→ Firewall issue (most common)
→ Router blocking device-to-device communication
