======================================================================
                        💼 HOW IT WORKS (WORLD-CLASS LOGIC)
======================================================================

PROCUREMENT WORKFLOW:

1. DEPARTMENT REQUESTS SUPPLIES
   - User creates Procurement Request
   - Lists items needed
   - Specifies quantities and estimates
   - Status: "Draft"

2. LEVEL 1 APPROVAL (Department Head/Manager)
   - Reviews request
   - Checks necessity and budget
   - Approves or rejects
   - Status: "Submitted" → "Admin Approved"

3. LEVEL 2 APPROVAL (Accounts Department) ← MAGIC HAPPENS HERE!
   - Accounts reviews for budget compliance
   - When approved, AUTOMATICALLY creates:
     
     a) ACCOUNTS PAYABLE (AP)
        - Records liability to vendor
        - Amount: Total procurement cost
        - Due Date: 30 days from approval
        - Status: Pending payment
     
     b) EXPENSE ENTRY
        - Recognizes expense (accrual accounting)
        - Category: Procurement Expenses
        - Status: Approved
        - Links to procurement request
     
     c) PAYMENT VOUCHER
        - Authorizes payment
        - Status: Ready for payment
        - Includes all details
        - Ready for payment processing
   
   Status: "Accounts Approved" → "Ready for Payment"

4. PAYMENT PROCESSING
   - Finance reviews payment voucher
   - Processes payment to vendor
   - Updates AP (marks as paid)
   - Updates voucher (marks as paid)
   - Status: "Payment Processed"

5. RECEIVING & COMPLETION
   - Goods received
   - Marked as complete
   - Status: "Received" → "Completed"


======================================================================
                        📚 ACCOUNTING PRINCIPLES USED
======================================================================

1. ACCRUAL ACCOUNTING:
   - Expense recognized when approved (not when paid)
   - Matches expenses to the period incurred

2. ACCOUNTS PAYABLE (LIABILITY):
   - Records what we owe to vendors
   - Tracked until paid
   - Proper aging analysis

3. EXPENSE RECOGNITION:
   - Expense recorded when procurement approved
   - Proper categorization
   - Budget tracking

4. PAYMENT AUTHORIZATION:
   - Payment voucher system
   - Approval workflow
   - Audit trail

5. SEGREGATION OF DUTIES:
   - Requester ≠ Approver
   - Approver ≠ Payment processor
   - Clear accountability chain

METHOD 2: Via Python Script (Programmatic)
-------------------------------------------
Create file: approve_procurement.py

```python
from hospital.models_procurement import ProcurementRequest
from hospital.procurement_accounting_integration import (
    auto_create_accounting_on_approval
)

# Get procurement request
pr = ProcurementRequest.objects.get(request_number='PR-202511-0001')

# Create accounting entries
result = auto_create_accounting_on_approval(pr)

if result:
    print(f"Success! Created:")
    print(f"  AP: {result['accounts_payable']}")
    print(f"  Expense: {result['expense']}")
    print(f"  Voucher: {result['payment_voucher']}")
```

METHOD 3: Automatic on Status Change
-------------------------------------
Add to your procurement approval view:

```python
from hospital.procurement_accounting_integration import (
    auto_create_accounting_on_approval
)

def approve_procurement_accounts(request, pr_id):
    pr = ProcurementRequest.objects.get(id=pr_id)
    
    # Approve
    pr.status = 'accounts_approved'
    pr.accounts_approved_by = request.user
    pr.accounts_approved_at = timezone.now()
    pr.save()
    
    # Auto-create accounting entries
    auto_create_accounting_on_approval(pr)
    
    return redirect('procurement_list')
```
