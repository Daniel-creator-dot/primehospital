                        🚀 QUICK START (5 MINUTES)
══════════════════════════════════════════════════════════════════════════════

Step 1: Update .env file
------------------------
DATABASE_URL=postgresql://user:password@192.168.1.20:5432/hms_production
DATABASE_SSL_MODE=require

(Replace with your actual database server details)


Step 2: Install PostgreSQL driver
----------------------------------
pip install psycopg2-binary


Step 3: Test connection
------------------------
python test_db_connection.py


Step 4: Run migrations
-----------------------
python manage.py migrate


Step 5: Start server
---------------------
python manage.py runserver

Done! Your HMS is now using a remote database! 🎉

══════════════════════════════════════════════════════════════════════════════
                         🔧 CONFIGURATION OPTIONS
══════════════════════════════════════════════════════════════════════════════

Available .env Settings:

DATABASE_URL
  Main database connection string
  Format: postgresql://user:password@host:port/database

DATABASE_SSL_MODE
  SSL security level: disable, allow, prefer, require, verify-ca, verify-full
  Default: prefer
  Recommended: require (production)

DATABASE_CONN_MAX_AGE
  Keep connections alive (seconds)
  Default: 600 (10 minutes)

DATABASE_CONN_HEALTH_CHECKS
  Check connection health before use
  Default: True

DATABASE_TIMEOUT
  Connection timeout (seconds)
  Default: 10

DATABASE_SSL_ROOT_CERT
  Path to CA certificate (optional)

DATABASE_SSL_CERT
  Path to client certificate (optional)

DATABASE_SSL_KEY
  Path to client key (optional)

══════════════════════════════════════════════════════════════════════════════
                        🆘 TROUBLESHOOTING QUICK REFERENCE
══════════════════════════════════════════════════════════════════════════════

Can't connect?
  1. Check DATABASE_URL format
  2. Verify database server is running
  3. Test network: ping <db-server-ip>
  4. Test port: telnet <db-server-ip> 5432
  5. Check firewall rules

Authentication fails?
  1. Verify username/password in .env
  2. Check pg_hba.conf on database server
  3. Ensure user has database privileges

SSL errors?
  1. Try DATABASE_SSL_MODE=disable (test only)
  2. Use DATABASE_SSL_MODE=require for production
  3. Check PostgreSQL ssl settings

Slow performance?
  1. Increase DATABASE_CONN_MAX_AGE
  2. Enable Redis caching
  3. Check network latency
  4. Optimize database queries
