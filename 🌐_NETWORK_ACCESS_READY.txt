YOUR ACCESS URL:

   http://192.168.0.102:8000

This URL works from:

========================================
QUICK SETUP (3 STEPS):
========================================

STEP 1: Start Docker Desktop
-----------------------------
- Make sure Docker Desktop is running
- Wait for it to fully start

STEP 2: Configure Windows Firewall
----------------------------------
Right-click: allow_port_8000_firewall.bat
Select: "Run as administrator"
Click "Yes" when prompted

OR manually:
1. Windows Defender Firewall -> Advanced Settings
2. Inbound Rules -> New Rule
3. Port -> TCP -> 8000 -> Allow

STEP 3: Start Services
-----------------------
docker-compose up -d

========================================
TEST ACCESS:
========================================

1. From your computer:
   Open: http://192.168.0.102:8000

2. From another device (phone, tablet, other computer):
   - Connect to same WiFi network
   - Open browser: http://192.168.0.102:8000

3. From secondary router network:
   - If router is in bridge/AP mode: Same URL works
   - If router has separate network: See setup below

========================================
SECONDARY ROUTER SETUP:
========================================

If you have a secondary router (enslaved router):

OPTION 1: Bridge/Access Point Mode (Easiest)
----------------------------------------------
1. Configure secondary router as Access Point
2. All devices become part of main network (192.168.0.x)
3. Access via: http://192.168.0.102:8000
4. Works from both routers automatically

OPTION 2: Router Mode with Port Forwarding
-------------------------------------------
1. Keep secondary router in router mode
2. Configure port forwarding:
   - External Port: 8000
   - Internal IP: 192.168.0.102
   - Internal Port: 8000
3. Access via: http://[secondary-router-ip]:8000

========================================
TROUBLESHOOTING:
========================================

Can't access from other devices?
→ Run allow_port_8000_firewall.bat as Administrator

Connection refused?
→ Check Docker Desktop is running
→ Check services: docker-compose ps

400 Bad Request?
→ Restart services: docker-compose restart web

Secondary router devices can't access?
→ Put router in bridge/AP mode
→ Or configure port forwarding
