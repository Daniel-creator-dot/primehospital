# 🔑 Render Environment Variables - Copy & Paste Guide

**Exactly what to paste into Render Dashboard → Environment Variables**

---

## 📍 How to Set Environment Variables in Render

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click on your **`hms-web`** service
3. Go to **"Environment"** tab (left sidebar)
4. Click **"Add Environment Variable"** for each variable below
5. Copy the **NAME** and paste your **VALUE**
6. Click **"Save Changes"** at the bottom

---

## ✅ REQUIRED Variables (Set These Now!)

Copy these exactly as shown. Replace the example values with YOUR actual values.

### 1. ALLOWED_HOSTS
```
NAME: ALLOWED_HOSTS
VALUE: hms-web.onrender.com
```
**⚠️ IMPORTANT**: Replace `hms-web` with YOUR actual Render service name!

**Example**: If your service is `primecare-hms`, use: `primecare-hms.onrender.com`

---

### 2. SITE_URL
```
NAME: SITE_URL
VALUE: https://hms-web.onrender.com
```
**⚠️ IMPORTANT**: Replace with your actual Render URL (same as ALLOWED_HOSTS but with `https://`)

---

### 3. EMAIL_HOST
```
NAME: EMAIL_HOST
VALUE: smtp.gmail.com
```
**Note**: Use `smtp.gmail.com` if using Gmail. For other providers:
- **Outlook/Hotmail**: `smtp-mail.outlook.com`
- **Yahoo**: `smtp.mail.yahoo.com`
- **SendGrid**: `smtp.sendgrid.net`

---

### 4. EMAIL_HOST_USER
```
NAME: EMAIL_HOST_USER
VALUE: youremail@gmail.com
```
**⚠️ Replace with YOUR actual email address**

---

### 5. EMAIL_HOST_PASSWORD
```
NAME: EMAIL_HOST_PASSWORD
VALUE: your-app-specific-password-here
```

**⚠️ CRITICAL**: For Gmail, use an **App Password**, NOT your regular password!

**How to Get Gmail App Password:**
1. Go to: https://myaccount.google.com/security
2. Enable **2-Step Verification** (if not already)
3. Search for "App passwords" or go to: https://myaccount.google.com/apppasswords
4. Select **"Mail"** and **"Other (Custom name)"** → Type "HMS"
5. Click **Generate**
6. Copy the 16-character password (looks like: `abcd efgh ijkl mnop`)
7. Paste it here (remove spaces): `abcdefghijklmnop`

---

### 6. EMAIL_PORT
```
NAME: EMAIL_PORT
VALUE: 587
```
**Note**: Use `587` for TLS (recommended). Use `465` for SSL.

---

### 7. EMAIL_USE_TLS
```
NAME: EMAIL_USE_TLS
VALUE: True
```

---

### 8. DEFAULT_FROM_EMAIL
```
NAME: DEFAULT_FROM_EMAIL
VALUE: noreply@yourhospital.com
```
**Note**: This is the "From" address users see. Can be anything like:
- `noreply@primecare.com`
- `support@yourhospital.com`
- `info@yourdomain.com`

---

### 9. SMS_API_KEY
```
NAME: SMS_API_KEY
VALUE: your-sms-notify-gh-api-key
```
**⚠️ Replace with YOUR SMS Notify GH API key**

**Don't have one?** Get it from: https://sms.smsnotifygh.com/

---

### 10. SMS_SENDER_ID
```
NAME: SMS_SENDER_ID
VALUE: PrimeCare
```
**Note**: This is what appears as the sender name. Max 11 characters.
Examples: `PrimeCare`, `YourClinic`, `Hospital`

---

## 🚫 DON'T Set These (Auto-Generated by Render)

These are **automatically set** by Render. Don't add them manually:

- ❌ **SECRET_KEY** - Render auto-generates this
- ❌ **DATABASE_URL** - Auto-linked from PostgreSQL service
- ❌ **REDIS_URL** - Auto-linked from Redis service
- ❌ **PORT** - Auto-set by Render

## 🎯 Step-by-Step: Setting Variables in Render

### Visual Guide

1. **Go to your service**
   - Render Dashboard → Select `hms-web`

2. **Click Environment tab**
   - Left sidebar → "Environment"

3. **Add first variable**
   - Click **"Add Environment Variable"**
   - In left field (NAME_OF_VARIABLE): Type `ALLOWED_HOSTS`
   - In right field (value): Type your Render URL (e.g., `hms-web.onrender.com`)
   - Click the checkmark or press Enter

4. **Add remaining variables**
   - Click **"Add Environment Variable"** again
   - Repeat for each variable above

5. **Save changes**
   - Scroll to bottom
   - Click **"Save Changes"** button
   - Render will automatically redeploy (takes 2-3 minutes)

## ⚡ Quick Method: Import from .env File

If you have many variables, you can import them:

1. Create a file called `.env` on your computer with this content:

```env
ALLOWED_HOSTS=your-service-name.onrender.com
SITE_URL=https://your-service-name.onrender.com
EMAIL_HOST=smtp.gmail.com
EMAIL_HOST_USER=youremail@gmail.com
EMAIL_HOST_PASSWORD=your-gmail-app-password
EMAIL_PORT=587
EMAIL_USE_TLS=True
DEFAULT_FROM_EMAIL=noreply@yourhospital.com
SMS_API_KEY=your-sms-api-key
SMS_SENDER_ID=PrimeCare
```

2. In Render Dashboard:
   - Go to Environment tab
   - Click **"Add from .env"**
   - Upload your `.env` file
   - Click **"Save Changes"**

## 🆘 Common Issues

### "ALLOWED_HOSTS" Error
**Error**: `DisallowedHost at /`

**Fix**: Make sure `ALLOWED_HOSTS` matches your exact Render URL
```
✅ Correct: hms-web.onrender.com
❌ Wrong: https://hms-web.onrender.com (no https://)
❌ Wrong: hms-web (no .onrender.com)
```

---

### Email Not Sending
**Error**: `Authentication failed`

**Fix**: Use Gmail App Password, not regular password
1. Go to: https://myaccount.google.com/apppasswords
2. Generate new app password
3. Copy the 16-character code (remove spaces)
4. Update `EMAIL_HOST_PASSWORD` with this code

---

### SMS Not Sending
**Error**: SMS not received

**Fix**: 
1. Verify `SMS_API_KEY` is correct
2. Check SMS balance at SMS Notify GH
3. Verify phone numbers include country code (e.g., +233...)

---

### Changes Not Taking Effect
**Issue**: Updated variable but app still uses old value

**Fix**:
1. Make sure you clicked **"Save Changes"**
2. Wait for automatic redeploy to complete (2-3 minutes)
3. Check "Logs" tab to verify redeploy finished
4. Hard refresh your browser (Ctrl+Shift+R)
