✅ CASHIER SESSION VALIDATION COMPLETE!
========================================

Cashiers must now account for denominations and match cash sales before closing sessions.

========================================
VALIDATIONS ADDED:
========================================

1. ✅ REQUIRED: Denomination Breakdown
   - Cashiers MUST count and enter all cash denominations
   - Cannot close session without denomination breakdown
   - System validates that breakdown totals match entered amount

2. ✅ REQUIRED: Denomination Total Match
   - Denomination breakdown total must match actual cash entered
   - Prevents errors in counting
   - Shows clear error if mismatch

3. ✅ REQUIRED: Cash Sales Match
   - System recalculates expected cash from all transactions before closing
   - Ensures expected cash matches actual cash sales
   - Validates opening cash + cash payments - cash refunds = expected cash

4. ✅ REQUIRED: Variance Explanation
   - If variance > 50 pesewas, cashier must provide explanation
   - If variance > 1% of expected, detailed notes required
   - All variances are logged in audit trail

5. ✅ ACCOUNTANT ACCESS TO CASHIER HUB
   - Accountants can now access cashier hub/dashboard
   - Access via: /hms/cashier/ or /hms/cashier/central/
   - Can view all cashier sessions and transactions
   - Can monitor cashier activities

========================================
CLOSING SESSION FLOW:
========================================

1. Cashier clicks "Close Session"
2. System shows expected cash (recalculated from transactions)
3. Cashier MUST:
   - Count all denominations (200, 100, 50, 20, 10, 5, 2, 1, 0.5, 0.2, 0.1, 0.05, 0.01)
   - Enter count for each denomination
   - System calculates total automatically
4. System validates:
   - Denominations entered ✓
   - Denomination total matches entered total ✓
   - Actual cash matches expected cash (with tolerance) ✓
5. If variance > 50 pesewas:
   - Cashier must provide explanation
   - Cannot close without notes
6. Session closes with full audit trail

========================================
BUTTON STATES:
========================================

- Disabled (Red): No denominations entered
- Warning (Yellow): Variance within tolerance but > 0
- Success (Green): Perfect match (within 1 pesewa)
- Error (Red): Large variance requiring explanation

========================================
ACCOUNTANT ACCESS:
========================================

Accountants can access:
- Cashier Hub: /hms/cashier/central/
- Cashier Dashboard: /hms/cashier/dashboard/
- View all sessions
- Monitor cashier activities
- Review cash reconciliations

Access is controlled via:
- user_has_cashier_access() function
- Includes: admin, accountant, cashier roles

========================================
AUDIT TRAIL:
========================================

Every session closure includes:
- Full denomination breakdown
- Expected vs Actual cash
- Variance analysis
- Timestamp and user information
- IP address and user agent
- Session duration
- All notes and explanations
