========================================
✅ CASHBOOK ERROR FIXED!
========================================

Fixed TypeError when accessing Cashbook add page in Django admin.

========================================
ERROR:
========================================

TypeError: '>=' not supported between instances of 'datetime.date' and 'NoneType'
Location: hospital/models_accounting_advanced.py, line 1163, in can_classify

This occurred when trying to access:
http://192.168.0.102:8000/admin/hospital/cashbook/add/

========================================
ROOT CAUSE:
========================================

The `can_classify()` method was trying to compare `today >= self.held_until`
when `self.held_until` could be `None` for new Cashbook entries that haven't
been saved yet (during form rendering in Django admin).

========================================
FIXES APPLIED:
========================================

1. ✅ Added None check in `can_classify()` method
   - Now returns `False` if `held_until` is `None`
   - Prevents comparison error

2. ✅ Made `held_until` field nullable
   - Changed from `DateField()` to `DateField(null=True, blank=True)`
   - More correct since the field is set in the `save()` method

3. ✅ Created and applied migration
   - Migration: `1038_fix_cashbook_held_until_nullable.py`

========================================
CODE CHANGES:
========================================

Before:
```python
def can_classify(self):
    """Check if entry can be classified to revenue (next day has arrived)"""
    today = timezone.now().date()
    return today >= self.held_until and self.status == 'pending'
```

After:
```python
def can_classify(self):
    """Check if entry can be classified to revenue (next day has arrived)"""
    if not self.held_until:
        return False
    today = timezone.now().date()
    return today >= self.held_until and self.status == 'pending'
```

Field definition:
```python
# Before
held_until = models.DateField(help_text="...")

# After
held_until = models.DateField(null=True, blank=True, help_text="...")
```

========================================
VERIFICATION:
========================================

The Cashbook add page should now work without errors:
http://192.168.0.102:8000/admin/hospital/cashbook/add/

The `can_classify_now` field in the admin form will show `False` for new
entries (which is correct - they can't be classified until saved and
the next day arrives).
