========================================

On System Boot:
1. Windows starts
2. Docker Desktop starts automatically (via scheduled task)
3. HMS startup script waits for Docker Desktop (up to 2 minutes)
4. Once Docker Desktop is ready, HMS services start automatically
5. Your server is accessible at http://192.168.0.102:8000

Total Time: 2-3 minutes after boot

========================================
TO DISABLE AUTO-START:
========================================

Option 1: Docker Desktop Settings
   - Open Docker Desktop
   - Settings -> General
   - Uncheck "Start Docker Desktop when you log in"

Option 2: Task Scheduler
   - Open Task Scheduler
   - Find: "Docker_Desktop_AutoStart" -> Disable
   - Find: "HMS_Docker_AutoStart" -> Disable

Option 3: Startup Folder
   - Delete: HMS_AutoStart.lnk from Startup folder
   - Location: C:\Users\bytz\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup

========================================
TROUBLESHOOTING:
========================================

Services not starting after boot?
   - Check Docker Desktop is set to auto-start
   - Check Task Scheduler for errors
   - Run START_EVERYTHING.bat manually

Docker Desktop not starting?
   - Check Docker Desktop settings
   - Verify scheduled task is enabled
   - Start Docker Desktop manually

Services start but can't access?
   - Run allow_port_8000_firewall.bat as Administrator
   - Check: docker-compose ps
   - Check: docker-compose logs web
