QUICK TEST GUIDE - DUPLICATE PREVENTION
═══════════════════════════════════════════════════════════════════

EASIEST TEST (Web Interface):
──────────────────────────────
1. Open your browser
2. Go to: http://localhost:8000/hms/patients/new/
   OR: http://localhost:8000/hms/patient-registration/
3. Fill in patient form:
   - First Name: Test
   - Last Name: Patient
   - Date of Birth: 1990-01-01
   - Phone: 0241234567
   - Email: test@example.com
4. Click "Register Patient"
5. Note the MRN created (e.g., PMC2025000123)

6. Go back to patient registration form
7. Fill in EXACT SAME information
8. Click "Register Patient" again

✅ EXPECTED: You should see error message:
   "⚠️ Duplicate patient detected! A patient with the same name..."

❌ IF DUPLICATE IS CREATED: The fix is not working


COMMAND LINE TEST:
──────────────────
Run this script to check for existing duplicates:

   python test_duplicate_prevention.py

This will:
- Check all patients for duplicates
- Show any duplicates found
- Tell you if system is working correctly


CHECK MIGRATION:
────────────────
Make sure the migration was applied:

   python manage.py showmigrations hospital | findstr 1043

Should show: [X] 1043_add_patient_duplicate_indexes


FIX EXISTING DUPLICATES:
────────────────────────
If duplicates are found, fix them:

   python manage.py fix_duplicates --dry-run    (see what will be fixed)
   python manage.py fix_duplicates --fix        (actually fix them)


TEST PHONE NUMBER NORMALIZATION:
─────────────────────────────────
1. Register patient with phone: 0241234567
2. Try to register again with: +233241234567
3. Try to register again with: 233241234567

All should be detected as duplicates (same number, different format)


TEST DOUBLE-CLICK PREVENTION:
──────────────────────────────
1. Fill out patient form
2. Rapidly click "Register Patient" button multiple times
3. Button should disable and show "Registering..."
4. Only ONE patient should be created
